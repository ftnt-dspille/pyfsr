pyfsr.api.picklists¶

Picklist resolution: name discovery, value lookup, IRI mapping.

Ported from the fsr-playbook-framework runner, de-coupled from its local sqlite reference DB. All discovery is live against the FortiSOAR REST API and cached in-process for the lifetime of the client.

The hard part FortiSOAR makes you do is turn a friendly picklist value (e.g. "High") into the IRI (/api/3/picklists/<uuid>) the API actually stores. This API resolves that, and can auto-discover which picklist a (module, field) pair binds to from the module’s field metadata.

Example

>>> client = demo_client()
>>> client.picklists.list()                       # all picklist names
['AlertStatus', 'Severity']
>>> [v["itemValue"] for v in client.picklists.values("AlertStatus")]
['Open', 'Investigating', 'Pending', 'Closed', 'Re-Opened']
>>> client.picklists.for_field("alerts", "status")  # the bound picklist name
'AlertStatus'
>>> client.picklists.resolve("High", picklist="Severity")
'/api/3/picklists/...'
>>> client.picklists.resolve_record_fields(
...     "alerts", {"name": "x", "severity": "High", "status": "Open"})
{'name': 'x', 'severity': '/api/3/picklists/...', 'status': '/api/3/picklists/...'}

resolve(value: str, *, picklist: str | None = None, module: str | None = None, field: str | None = None) → str | None[source]¶

Resolve a friendly value (e.g. "High") to its picklist IRI.

Provide either an explicit picklist name, or (module, field) to auto-discover it. Already-IRI strings pass through unchanged. Returns None if the value isn’t found in the resolved picklist.

options(picklist_name: str) → list[str][source]¶

The valid friendly values (itemValues) of a picklist — what an AI should choose from. Cached via values().

resolve_record_fields(module: str, fields: dict[str, Any], *, strict: bool = False, report: list | None = None) → dict[str, Any][source]¶

Return a copy of fields with picklist-typed values mapped to IRIs.

Only fields the module flags as picklist-backed are touched, and only when their value is a friendly string (not already an IRI).

Friendly feedback on a miss (a value not in the picklist):

  • strict=True raises PicklistResolutionError naming the field, bad value, and the valid options.

  • pass a list as report to collect misses as {field, value, picklist, valid_values} without raising.

  • by default the original value is left in place (back-compatible).

validate_record_fields(module: str, fields: dict[str, Any]) → list[dict[str, Any]][source]¶

Dry-run picklist resolution: return the misses without mapping/writing.

Runs the same resolution resolve_record_fields() would, but discards the resolved dict and returns only the unresolved picklist values as [{field, value, picklist, valid_values}, ...] — empty list means every picklist field resolves cleanly. Lets a caller validate its mappings before committing any write.

Example

>>> client = demo_client()
>>> misses = client.picklists.validate_record_fields(
...     "alerts", {"severity": "Critical", "status": "Bogus"})
>>> [m["field"] for m in misses]
['status']
create_picklist(name: str, *, system: bool = False, options: list[str] | list[dict[str, Any]] | None = None) → pyfsr.models._system.PicklistName[source]¶

Create a picklist list (optionally with initial options).

Parameters:

  • name – the list’s friendly name (unique instance-wide — a duplicate 409s; use get_or_create_picklist() for idempotency).

  • system – the system flag (custom lists are False).

  • options – optional initial option labels. A bare string becomes an item with auto orderIndex; a dict can carry {"value", "color", "order"} (order overrides the position). Created in order.

Returns:

The created PicklistName (with its options embedded when options was given).

Raises:

APIError – on a 409 duplicate name (UniqueConstraintViolationException).

get_or_create_picklist(name: str, *, system: bool = False, options: list[str] | list[dict[str, Any]] | None = None) → tuple[pyfsr.models._system.PicklistName, bool][source]¶

Idempotently ensure picklist name exists; return (list, created).

If the list already exists, its options are not modified (only the list is ensured) — use add_option() to append. Returns created=True only when the list was newly created.

add_option(picklist: str, value: str, *, color: str | None = None, order: int | None = None) → pyfsr.models._system.PicklistItem[source]¶

Add an option (item) to an existing picklist.

Parameters:

  • picklist – the target list — a name, a list IRI (/api/3/picklist_names/<uuid>), or a bare uuid.

  • value – the option’s friendly label (itemValue).

  • color – optional hex color (e.g. "#FF0000").

  • order – the orderIndex; defaults to appending after the list’s current highest index.

Returns:

The created PicklistItem (its iri is what a record stores for this value).

Raises:

ValueError – if picklist can’t be resolved to a list IRI.

get_or_create_option(picklist: str, value: str, *, color: str | None = None, order: int | None = None) → tuple[pyfsr.models._system.PicklistItem, bool][source]¶

Idempotently ensure option value exists in picklist; return (item, created).

If an option with the same itemValue already exists, it is returned unchanged (its color/orderIndex are not modified — only presence is ensured). Returns created=True only when the option was newly added.

Parameters:

  • picklist – the target list — a name, IRI, or bare uuid.

  • value – the option’s friendly label (itemValue).

  • color – hex color for a newly-created option (ignored if existing).

  • order – orderIndex for a newly-created option (ignored if existing).

Returns:

(PicklistItem, created) — the existing item with created=False, or the newly-created item with created=True.

remove_option(picklist: str | None = None, *, value: str | None = None, item: str | None = None, missing_ok: bool = True) → bool[source]¶

Remove an option from a picklist.

Identify the item either by its value within picklist (resolved to an IRI) or directly by its item IRI/uuid. Returns True if an item was deleted; with missing_ok=True (default) an absent item returns False rather than raising.

Raises:

  • ValueError – if neither item nor value is given, or value is given without picklist.

  • ResourceNotFoundError – if the item is absent and missing_ok=False.

remove_picklist(picklist: str, *, missing_ok: bool = True) → bool[source]¶

Delete a picklist list and (per the platform) cascade-delete its items.

Parameters:

  • picklist – the list — a name, IRI, or bare uuid.

  • missing_ok – when True (default), an absent list returns False instead of raising.

Returns:

True if a list was deleted.

Raises:

ResourceNotFoundError – if the list is absent and missing_ok=False.

get_picklist(picklist: str, *, relationships: bool = True) → pyfsr.models._system.PicklistName | None[source]¶

Fetch one picklist list by name, IRI, or uuid (None if absent).

With relationships=True (default) the options are embedded under picklists (so .items is populated); without it the list is returned bare. Resolves a name to its IRI via the bulk name index first.
